# Plan: Fix Layout & Styling Issues After main.css Update

## Objective

Fix the broken layout and styling caused by the updated `app/assets/css/main.css` file.

## Root Cause Analysis

The added CSS in the update is **conflicting with Nuxt UI's base styles**:

1. **Universal reset** (`*, *::before, *::after { margin: 0; padding: 0; }`) is clobbering Nuxt UI's reset/Preflight styles
2. **Body font override** (`font-family: var(--font-body)`) conflicts with Nuxt UI which uses `--font-sans` via Tailwind
3. **Heading styles** (`h1-h4 { font-family: var(--font-display) }`) may conflict with Nuxt UI component styles
4. **The `:root` block** with `--space-*` and `--radius-*` variables is unused by components (they use Tailwind classes)

## Implementation Steps

### Step 1: Remove Conflicting Base Styles from `app/assets/css/main.css`

Edit `app/assets/css/main.css` and:

1. **Remove the entire `:root` block** (lines 37-55) - these variables aren't used
2. **Remove the universal reset** (lines 58-64) - Nuxt UI/Tailwind Preflight already handles this
3. **Remove the `body` styles** (lines 71-78) - let Nuxt UI use `--font-sans` from `@theme`
4. **Remove the heading styles** (lines 81-88) - components use `font-display` class directly
5. **Remove the `.eyebrow` class** (lines 90-97) - can be added back as a Tailwind component if needed

The file should only contain:

```css
@import "tailwindcss";
@import "@nuxt/ui";

@theme {
  /* Fonts */
  --font-sans: "Poppins", sans-serif;
  --font-display: "Roobert PRO Medium", "Roobert PRO", -apple-system, BlinkMacSystemFont, sans-serif;
  --font-body: "Noto Sans", -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;

  /* Brand Colors */
  --color-accent: #5b76fe;
  --color-accent-pressed: #2a41b6;
  --color-success: #00b473;
  --color-muted: #555a6a;
  --color-ring: #e0e2e8;
  --color-border: #c7cad5;
  --color-bg: #ffffff;
  --color-fg: #1c1c1e;

  /* Surface Colors */
  --color-coral-light: #ffc6c6;
  --color-coral-dark: #600000;
  --color-teal-light: #c3faf5;
  --color-teal-dark: #187574;

```

### Step 2: Fix Font Loading with Matching Alternatives

The fonts "Roobert PRO" and "Noto Sans" need to be replaced with available alternatives:

**Recommended Font Matches:**

| Original Font        | Replacement            | Reasoning                                                                                                   |
| -------------------- | ---------------------- | ----------------------------------------------------------------------------------------------------------- |
| "Roobert PRO Medium" | **Inter** (500 weight) | Inter is a modern, clean sans-serif designed for UI, closely matching Roobert's proportions and readability |
| "Noto Sans"          | **Noto Sans**          | Available on Google Fonts - import directly                                                                 |
| "Poppins"            | **Poppins**            | Already available on Google Fonts                                                                           |

**Update `main.css`** - Add Google Fonts import at the top:

```css
@import url("https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Noto+Sans:wght@400;600;700&family=Poppins:wght@400;600;700&display=swap");

@import "tailwindcss";
@import "@nuxt/ui";

@theme {
  /* Fonts - using available Google Fonts */
  --font-sans: "Poppins", system-ui, sans-serif;
  --font-display: "Inter", system-ui, sans-serif;
  --font-body: "Noto Sans", system-ui, sans-serif;

  /* Rest of theme... */
}
```

**Why Inter for Roobert PRO:**

- Both are modern, geometric sans-serifs
- Inter has similar x-height and character proportions
- Inter 500 weight closely matches "Roobert PRO Medium"
- Inter is optimized for UI/screen display like Roobert

### Step 3: Verify Tailwind Classes Are Generated

After cleaning up the CSS, the `@theme` variables should generate utility classes:

- `--color-accent` → `bg-accent`, `text-accent`, `border-accent`, etc.
- `--color-muted` → `text-muted`, `bg-muted`, etc.
- `--font-display` → `font-display` class
