# write-ux-ui-flow-and-copy

Act as an expert UX Writer. Based on the provided sitemap, generate the top-to-bottom UI structure and final copy for the {{PAGE}} page.

Format the output as a sequential list of UI components (e.g., Header, Hero Section, Feature Grid, Call-to-Action). For each component, provide the complete, final text—strictly avoid placeholder text like "Lorem Ipsum".
 
**Copywriting Guidelines:**
*   Make it compelling, concise, and emotionally resonant.
*   Focus on user benefits and driving clear actions.
*   Ensure the text is highly skimmable (use headings, short sentences, and bullet points).
*   Use the Problem-Agitation-Solution framework
*   Focus on specific benefits, not vague features
*   Create urgency and emotional resonance
*   Use short, powerful sentences
*   Cut unnecessary words and business jargon
*   Make CTAs clear, compelling, and action-oriented
*   Enhance the problem/agitation sections to create more urgency
*   Ensure all benefits are concrete, not abstract
 
**Constraints:**
*   Include only structural building blocks and text content.
*   Strictly exclude all code, CSS, styling instructions, or visual UI specifications. 


## Steps

1. Analyze the provided sitemap that can be found in your memory bank to understand the context and flow of the {{PAGE}} page. Review your knowledge base for any relevant information about the target audience, product/service, and key messaging that should be incorporated into the copy.
2. Format your output as a sequential list of UI components from top to bottom (e.g., Header, Hero Section, Feature Grid, Call-to-Action).
3. Draft the complete, final text for each component. You must strictly avoid using placeholder text like "Lorem Ipsum".
4. Apply the following **Copywriting Guidelines** to all text:
   * Make it compelling, concise, and emotionally resonant.
   * Focus on user benefits and driving clear actions.
   * Ensure the text is highly skimmable by utilizing headings, short sentences, and bullet points.
5. Review your final output against these strict **Constraints**:
   * Include *only* structural building blocks and text content.
   * Strictly exclude all code, CSS, styling instructions, or visual UI specifications.
